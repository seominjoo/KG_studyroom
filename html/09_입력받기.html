# input 태그를 이용해 사용자로부터 다양한 입력을 받을 수 있다.

  • 사용자의 입력들은 폼(form)단위로 묶어놓아야 한다.
  • 해당 form 내부에서 submit()이 발생하면 데이터를 전송된다.
  • input태그를 form태그의 외부에서 추가할 수도 있다.
  • 데이터 전송 시 input태그의 name속성이 key역할을 한다.
  • 데이터 전송 시 input태그에 사용자가 입력한 내용이 value역할을 한다.
  • input태그의 value속성이 value역할을 할 때도 있다.
Type 모양 설명
type="text" 일반적인 텍스트를 입력하는 칸
type="password" 입력하는 내용이 가려지는 칸
type="submit" 사용자가 input태그에 입력해놓은 데이터를 전송하는 버튼
type="reset" 사용자가 input태그에 입력해놓은 데이터를 초기화하는 버튼
type="radio" 여러가지 중 하나만 선택할 수 있는 버튼
type="checkbox" 동시에 여러가지를 선택할 수 있는 버튼
type="email" 해당 텍스트가 이메일인지 검사해준다.
<select> country: 사용자가 여러 옵션 중 하나를 선택할 수 있다.
<textarea> 많은 양의 텍스트를 입력할 때 사용하는 태그

# 유용한 input 태그의 속성들

  • checked : 페이지가 로드되었을때 미리 체크 해놓는다.(radio or checkbox)
  • selected : 미리 보여준다.(select)
  • disabled : 해당 필드를 사용할 수 없계 된다.(전송에서 제외된다)
  • readonly : 해당 필드를 수정할 수 없게 된다.(전송에는 포함된다)
  • required : 해당 필드의 값을 반드시 채워야 전송된다.
  • pattern : 해당 필드에 정규표현식 검사를 할 수 있다.
  • size : 필드의 크기(너비)를 설정할 수 있다.
  • value : 값을 지정해놓을 수 있다.(기본값 설정)
  • maxlength : 최대 길이를 설정
  • min : 최소값 설정
  • max : 최대값 설정
  • step : 값의 단위를 설정
